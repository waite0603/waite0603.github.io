npm install 遇到的一系列问题

发表于2022-02-01|更新于2022-02-01
|字数总计:741|阅读时长:3分钟|阅读量:

依赖冲突问题

  • 报错
  • 报错依赖版本不一致 安装时忽略所有peerDependencies
  • 还是报错
  • 清除缓存再试一次

node和node-sass版本

nodes-sass releases地址：https://github.com/sass/node-sass/releases
nodejs releases地址：https://nodejs.org/zh-cn/download/releases/

进入网站查看两者互相兼容的版本

npm与node.js版本不兼容

  • 报错
  • 重新安装NodeJs，安装文件自行到官网下载，官网地址：https://nodejs.org/zh-cn/download/
  • 不要随便升级 npm

node或npm提示不是命令or程序

  • 检查 npm 或者 node 是否成功安装
  • 以管理员身份运行编译器
  • Vscode 在管理员状态下 新建终端
  • WebStorm Settings => Tools => Terminal 更换 ShellPath
  • 如果都实现不了, 尝试一下重启电脑

vue 提示不是命令or程序

  • @vue/cli的安装
  • 检查@vue/cli是否安装成功
  • 如果提示：’vue’ 不是内部或外部命令，也不是可运行的程序 该如何解决

npm config list 查看一下npm 的配置信息，如下

  • 找到 prefix: 一行进入该文件夹, 看看有没有vue.cmd的文件，然后把这个路径添加的系统环境变量的path里面
  • 重新打开cmd窗口执行vue –version，看看是否成功
